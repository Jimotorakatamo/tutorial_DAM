Aprende HTML Moderno

HTML es el lenguaje fundamental de la web. Con él podrás crear la estructura básica de cualquier página web. En este tutorial, aprenderás desde los conceptos básicos hasta las características más modernas de HTML5.

Estructurado

Organiza el contenido de forma lógica

Accesible

Compatible con lectores de pantalla

Responsivo

Se adapta a todos los dispositivos

¿Para qué sirve HTML?

HTML define la estructura y el significado del contenido web. Con él describes encabezados, párrafos, formularios, tablas y cualquier bloque que después podrás enriquecer con CSS y JavaScript.

Web corporativa

Landing pages, portfolios y sitios institucionales con jerarquías claras.

Catálogos online

Estructura fichas de producto, listas de precios y formularios de compra.

Documentación

Guías técnicas, wikis y recursos educativos accesibles.

Paso a paso

  1. Define el objetivo: lista qué secciones necesita la página (hero, servicios, contacto...).
  2. Traza el esqueleto: crea la estructura base con <header>, <main> y <footer>.
  3. Añade semántica: usa etiquetas específicas como <article>, <section> o <nav>.
  4. Revisa accesibilidad: comprueba orden de encabezados, textos alternativos e interacciones de teclado.
<main>
    <section class="hero" role="banner">
        <h1>Academia DAM</h1>
        <p>Formación práctica en desarrollo multiplataforma.</p>
        <a class="cta" href="#cursos">Ver cursos</a>
    </section>

    <section id="cursos" aria-labelledby="titulo-cursos">
        <h2 id="titulo-cursos">Oferta formativa</h2>
        <article class="card">
            <h3>Programación</h3>
            <p>Algoritmos, POO y patrones para aplicaciones empresariales.</p>
        </article>
    </section>
</main>

Estructura Básica

Todo documento HTML tiene una estructura fundamental que define su organización. Aquí tienes un ejemplo de la estructura básica:

DOCTYPE

Define el tipo de documento

HTML

Elemento raíz del documento

HEAD

Metadatos y recursos

BODY

Contenido visible

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <title>Mi Primera Página</title>
</head>
<body>
    <h1>Hola, Mundo</h1>
    <p>Este es mi primer documento HTML.</p>
</body>
</html>

Multimedia en HTML

HTML5 facilitó enormemente la inserción de contenido multimedia sin necesidad de plugins externos.

Metaetiquetas y SEO

Las metaetiquetas en el <head> proporcionan información crucial para los navegadores y los motores de búsqueda (SEO).

<head>
    <!-- Título de la página (muy importante para SEO) -->
    <title>Título Descriptivo de la Página</title>

    <!-- Descripción para los motores de búsqueda -->
    <meta name="description" content="Una descripción concisa de tu página (aprox. 155 caracteres).">

    <!-- Palabras clave (menos relevante hoy en día, pero no hace daño) -->
    <meta name="keywords" content="html, tutorial, web, desarrollo">

    <!-- Protocolo Open Graph para redes sociales (Facebook, LinkedIn...) -->
    <meta property="og:title" content="Título para Redes Sociales">
    <meta property="og:description" content="Descripción que aparecerá al compartir.">
    <meta property="og:image" content="https://ejemplo.com/imagen-preview.jpg">
    <meta property="og:url" content="https://ejemplo.com/pagina.html">
</head>

Atributos Avanzados

Además de los atributos comunes, HTML ofrece otros para funcionalidades más específicas.

Atributos data-*

Permiten almacenar información extra y personalizada en cualquier elemento HTML, que luego puede ser fácilmente accedida con JavaScript.

<div id="usuario" data-id="12345" data-role="admin">
    John Doe
</div>
<script>
    const usuarioDiv = document.getElementById('usuario');
    console.log(usuarioDiv.dataset.id); // "12345"
    console.log(usuarioDiv.dataset.role); // "admin"
</script>

Gráficos en la Web

HTML5 introdujo dos potentes formas de crear gráficos directamente en el navegador.

Canvas

Un lienzo para dibujar gráficos 2D y 3D dinámicamente con JavaScript. Ideal para animaciones, juegos y visualización de datos en tiempo real.

SVG (Scalable Vector Graphics)

Gráficos vectoriales basados en XML. Son escalables sin perder calidad y se pueden manipular con CSS y JavaScript. Perfectos para logos e iconos.

<!-- Un círculo simple con SVG -->
<svg width="100" height="100">
   <circle cx="50" cy="50" r="40" stroke="black" stroke-width="3" fill="red" />
</svg>

<!-- Un lienzo para dibujar con JavaScript -->
<canvas id="miCanvas" width="200" height="100" style="border:1px solid #000;"></canvas>

¡A Practicar!

Copia el código del círculo SVG en tu página. Intenta cambiar el color de relleno (fill) a "blue" y el radio (r) a "20".

APIs de HTML5

HTML5 no es solo etiquetas, también es un conjunto de APIs de JavaScript que dan superpoderes al navegador.

Web Storage

Permite almacenar datos en el navegador del usuario con localStorage (persistente) y sessionStorage (por sesión).

Geolocation

Permite al usuario compartir su ubicación con la aplicación web.

Web Workers

Ejecuta scripts en segundo plano sin congelar la interfaz de usuario.

// Ejemplo de uso de Local Storage
localStorage.setItem('nombreUsuario', 'Ana');
const usuario = localStorage.getItem('nombreUsuario');
console.log(usuario); // "Ana"

// Ejemplo de uso de Geolocation
navigator.geolocation.getCurrentPosition(pos => {
    console.log(`Latitud: ${pos.coords.latitude}, Longitud: ${pos.coords.longitude}`);
});

¡A Practicar!

Abre la consola de desarrollador de tu navegador (F12), ve a la pestaña "Consola" y pega el código de localStorage. Luego, recarga la página y comprueba en la pestaña "Aplicación" -> "Almacenamiento local" que el dato sigue ahí.

Accesibilidad y ARIA

Una página HTML completa debe ser usable por el mayor número posible de personas. Las etiquetas semánticas, junto con atributos ARIA, ayudan a los lectores de pantalla a interpretar la interfaz.

Roles y landmarks

Define regiones como banner, navigation o main para orientar dispositivos de asistencia.

Navegación por teclado

Gestiona el foco con tabindex y usa aria-live para avisos dinámicos.

Texto alternativo

Describe imágenes, iconos y controles visuales con alt o aria-label.

<header role="banner">
    <h1>Portal DAM</h1>
</header>

<nav aria-label="Navegación principal" role="navigation">
    <button aria-expanded="false" aria-controls="menu" aria-label="Abrir menú">
        <i class="fa-solid fa-bars" aria-hidden="true"></i>
    </button>
    <ul id="menu" hidden>
        <li><a href="#html">HTML</a></li>
        <li><a href="#css">CSS</a></li>
    </ul>
</nav>

<main role="main">
    <section aria-labelledby="seccion-form">
        <h2 id="seccion-form">Formulario de alta</h2>
        <form aria-describedby="ayuda-form">
            <label for="email">Correo institucional</label>
            <input id="email" type="email" required aria-required="true">
            <span id="ayuda-form" class="visually-hidden">El correo debe terminar en @centro.edu</span>
        </form>
    </section>
</main>

¡A Practicar!

Revisa cualquier formulario que hayas creado y añade etiquetas label, atributos aria-describedby y mensajes para usuarios con lector de pantalla. Comprueba con la tecla Tab que el foco recorre los campos en orden lógico.

Plantillas y Componentes Reutilizables

Con HTML puedes crear bloques reutilizables usando elementos como <template> o personalizando etiquetas con Web Components.

<template>

Define fragmentos ocultos listos para clonar desde JavaScript.

Slots

Los Web Components permiten insertar contenido dinámico dentro de plantillas encapsuladas.

Includes del servidor

En proyectos grandes reutiliza cabeceras mediante frameworks o motores de plantillas.

<template id="card-template">
    <article class="curso">
        <h3 class="curso__titulo"></h3>
        <p class="curso__descripcion"></p>
    </article>
</template>

<section id="cursos"></section>

<script>
const cursos = [
    { titulo: 'Interfaces Web', descripcion: 'HTML5, componentes web y accesibilidad.' },
    { titulo: 'Programación', descripcion: 'Java y patrones orientados a objetos.' }
];

const tpl = document.getElementById('card-template');
const contenedor = document.getElementById('cursos');

cursos.forEach(curso => {
    const fragmento = tpl.content.cloneNode(true);
    fragmento.querySelector('.curso__titulo').textContent = curso.titulo;
    fragmento.querySelector('.curso__descripcion').textContent = curso.descripcion;
    contenedor.appendChild(fragmento);
});
</script>

Crea un <template> para mostrar fichas de asignaturas del ciclo DAM. Añade al menos tres asignaturas con su descripción y genera el listado desde JavaScript.

Proyecto Integrador

Consolida lo aprendido construyendo una landing para un ciclo formativo DAM que combine estructura semántica, formularios y multimedia.

  1. Estructura general: usa header, main, section, aside y footer para dividir el contenido.
  2. Sección de módulos: crea una tabla o cards para listar asignaturas con horas y profesores.
  3. Testimonios: agrega un <blockquote> con citas de alumnado, acompañadas de imágenes accesibles.
  4. Formulario de inscripción: solicita nombre, email, turno preferido y acepta la política de datos mediante un checkbox obligatorio.
  5. Llamada a la acción: coloca un botón que enlace a la página del centro con atributos rel="noopener" y target="_blank".

¡Entrega opcional!

Publica tu landing en GitHub Pages o en Netlify. Comparte el enlace con tus compañeros para recibir feedback y proponte añadir animaciones CSS en la siguiente fase.

Validación y Mejores Prácticas

Antes de dar por finalizado un documento HTML conviene pasar por un checklist que garantice calidad y mantenimiento.

  • Valida el marcado en validator.w3.org para detectar etiquetas mal anidadas o atributos incorrectos.
  • Activa Lighthouse (Chrome DevTools) y revisa secciones de accesibilidad y SEO.
  • Comprueba que todos los enlaces funcionan y que las rutas de imágenes son relativas cuando corresponda.
  • Asegúrate de que los formularios incluyen estados de error visibles y mensajes de ayuda.
  • Incluye comentarios puntuales para secciones complejas, pero evita llenar el documento con anotaciones innecesarias.

js/modulos.js 
const parser = new DOMParser();
const lista = document.querySelector('[data-lista]');

async function cargarModulos() {
    const respuesta = await fetch('datos/modulos.xml');
    const texto = await respuesta.text();
    const xml = parser.parseFromString(texto, 'application/xml');
    const modulos = [...xml.querySelectorAll('modulo')];

    lista.innerHTML = modulos.map(modulo => `
        <li>
            <span>${modulo.getAttribute('nombre')}</span>
            <small>${modulo.querySelector('horas').textContent} h</small>
        </li>
    `).join('');
}

cargarModulos();
