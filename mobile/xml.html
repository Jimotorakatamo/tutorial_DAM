Aprende XML

XML (eXtensible Markup Language) es un lenguaje de marcado diseñado para almacenar y transportar datos. Es flexible, legible y ampliamente utilizado en la web y aplicaciones empresariales.

Organización jerárquica

Universal

Estándar internacional

Extensible

Personalizable

<?xml version="1.0" encoding="UTF-8"?>

    ¡Bienvenido al mundo XML!
</mensaje>

¿Para qué sirve XML?

XML modela datos estructurados y transportables entre sistemas heterogéneos. Es extensible, legible por humanos y soportado en integraciones empresariales desde hace décadas.

Integración empresarial

Mensajería entre ERP, CRM y plataformas de logística vía SOAP o colas.

Configuraciones

Define pipelines de despliegue, reglas de compilación o políticas de apps.

Intercambio de contenidos

Feeds RSS/Atom, catálogos de productos y documentos normalizados.

Paso a paso

  1. Define el dominio: enumera entidades, atributos y relaciones clave.
  2. Diseña la jerarquía: elige un elemento raíz y anida nodos según dependencia.
  3. Valida la estructura: crea un DTD/XSD para garantizar consistencia entre equipos.
  4. Automatiza la transformación: aplica XPath o XSLT para convertir los datos en vistas, reportes o JSON.

    </cliente>

Tu primer documento XML

Sigue este flujo cada vez que necesites definir datos estructurados. Mantiene tu XML legible y listo para validarse.

Pasos básicos

  1. Declara la cabecera: especifica versión y codificación para evitar problemas de caracteres.
  2. Elige un elemento raíz: debe describir el conjunto completo de datos (ej. <catalogo>).
  3. Define nodos hijos: reutiliza nombres coherentes y anida sólo cuando tenga sentido.
  4. Añade atributos: aporta metadatos compactos como identificadores, unidades o estados.
<?xml version="1.0" encoding="UTF-8"?>
<catalogo>
    <elemento>...</elemento>
</catalogo>

Sintaxis y Reglas

Un documento XML "bien formado" debe seguir reglas estrictas para ser procesado correctamente.

Reglas fundamentales

  • Debe tener un único elemento raíz.
  • Todas las etiquetas deben tener su correspondiente etiqueta de cierre.
  • Las etiquetas son sensibles a mayúsculas y minúsculas (<Libro> ≠ <libro>).
  • Los elementos deben estar anidados correctamente, sin solaparse.
  • Los valores de los atributos deben estar siempre entre comillas.

Validación: DTD y XSD

Para que un documento XML sea "válido", debe cumplir con una estructura definida, ya sea mediante un DTD o un XML Schema (XSD).

DTD

Document Type Definition - Define la estructura legal y los elementos

XSD

XML Schema Definition - Más potente, con tipos de datos y restricciones

Ejemplo DTD

<!ELEMENT catalogo (modulo+)>








'>
                
<!ELEMENT catalogo (modulo+)>
<!ATTLIST catalogo cursos CDATA #REQUIRED>

<!ELEMENT modulo (titulo, profesor, descripcion?)>
<!ATTLIST modulo codigo CDATA #REQUIRED
                  horas CDATA #REQUIRED
                  curso CDATA #IMPLIED>

<!ELEMENT titulo (#PCDATA)>
<!ELEMENT profesor (#PCDATA)>
<!ATTLIST profesor email CDATA #IMPLIED>
<!ELEMENT descripcion (#PCDATA)>

Ejemplo XSD

<xs:schema xmlns:xs="http://www.w3.org/2001/XMLSchema" 
           elementFormDefault="qualified">
    <xs:element name="catalogo">
        <xs:complexType>
            <xs:sequence>
                <xs:element ref="modulo" maxOccurs="unbounded"/>
            </xs:sequence>
            <xs:attribute name="cursos" type="xs:string" use="required"/>
        </xs:complexType>
    </xs:element>

    <xs:element name="modulo">
        <xs:complexType>
            <xs:sequence>
                <xs:element name="titulo" type="xs:string"/>
                <xs:element name="profesor" type="xs:string"/>
                <xs:element name="descripcion" type="xs:string" minOccurs="0"/>
            </xs:sequence>
            <xs:attribute name="codigo" type="xs:string" use="required"/>
            <xs:attribute name="horas" type="xs:positiveInteger" use="required"/>
            <xs:attribute name="curso" type="xs:string"/>
        </xs:complexType>
    </xs:element>
</xs:schema>

Validar desde consola

Usa herramientas como xmllint o xmlstarlet para validar tus documentos:

# Validar con DTD
xmllint --noout --dtdvalid catalogo.dtd catalogo.xml

# Validar con XSD
xmlstarlet val --err --xsd catalogo.xsd catalogo.xml

XPath: Consultando en XML

XPath es un lenguaje que permite navegar a través de los elementos y atributos de un documento XML para seleccionar nodos específicos.

Expresiones comunes

  • /biblioteca/libro[@categoria='ficcion'] → Selecciona todos los libros de ficción
  • //libro[precio > 25] → Selecciona libros con precio mayor a 25
  • //libro/titulo → Selecciona todos los títulos de los libros
  • //libro[1] → Selecciona el primer libro
  • //libro[last()] → Selecciona el último libro
/biblioteca/libro[@categoria="ficcion"]
//libro[precio > 25]
//libro/titulo
count(//libro)
string(//libro[1]/titulo)

XSLT: Transformando XML

XSLT (eXtensible Stylesheet Language Transformations) se usa para transformar un documento XML en otro formato, como HTML, PDF o texto plano.

<xsl:stylesheet version="1.0" 
                xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
    <xsl:template match="/">
        <html><body>
            <h2>Mis Libros</h2>
            <xsl:for-each select="biblioteca/libro">
                <p><xsl:value-of select="titulo"/> 
                   por <xsl:value-of select="autor"/></p>
            </xsl:for-each>
        </body></html>
    </xsl:template>
</xsl:stylesheet>

Uso práctico

XSLT es ideal para generar reportes HTML automáticamente desde datos XML, transformar feeds RSS en páginas web, o convertir estructuras XML de un formato a otro.

Namespaces

Los namespaces se utilizan para evitar conflictos de nombres entre elementos que se llaman igual pero tienen diferente significado. Se definen con el atributo xmlns.

<root>
    <h:table xmlns:h="http://www.w3.org/TR/html4/">
        <h:tr><h:td>Celda HTML</h:td></h:tr>
    </h:table>

    <f:table xmlns:f="https://www.w3schools.com/furniture">
        <f:name>Mesa</f:name>
        <f:width>80</f:width>
    </f:table>
</root>

¿Por qué usar namespaces?

Cuando trabajas con múltiples esquemas XML o combinas documentos de diferentes fuentes, los namespaces previenen conflictos entre elementos con el mismo nombre pero diferente propósito.

Integración con Java y Python

En el ciclo DAM es común procesar XML desde servidores Java o scripts Python.

Java

Bibliotecas como JAXB o DOM Parser permiten leer y escribir XML fácilmente

Python

Módulos estándar como xml.etree.ElementTree manejan árboles XML

Ejemplo en Java

// Java + JAXB
@XmlRootElement(name = "curso")
public class Curso {
    @XmlElement private String nombre;
    @XmlElement private int horas;
}

JAXBContext contexto = JAXBContext.newInstance(Curso.class);
Curso curso = (Curso) contexto.createUnmarshaller()
                .unmarshal(new File("curso.xml"));

Ejemplo en Python

import xml.etree.ElementTree as ET

tree = ET.parse("curso.xml")
root = tree.getroot()

nombre = root.findtext("nombre")
horas = root.findtext("horas")
print(f"Curso: {nombre} ({horas}h)")

XML en APIs y servicios

Algunas APIs REST o SOAP todavía consumen y responden en XML. Saber construir peticiones bien formadas es clave para integraciones empresariales.

Petición HTTP con XML

POST /api/matriculas HTTP/1.1
Host: api.centro.es
Content-Type: application/xml
Accept: application/xml

<matricula>
    <alumno id="A-102">
        <nombre>Sara Martín</nombre>
        <curso>DAM2</curso>
    </alumno>
    <modulo codigo="PRO"/>
    <modulo codigo="INT"/>
</matricula>

Consumir desde JavaScript

const cuerpo = `<matricula>...</matricula>`;

const respuesta = await fetch("https://api.centro.es/api/matriculas", {
    method: "POST",
    headers: {
        "Content-Type": "application/xml",
        "Accept": "application/xml"
    },
    body: cuerpo
});

const texto = await respuesta.text();
const xml = new DOMParser().parseFromString(texto, "application/xml");
const estado = xml.querySelector("estado").textContent;
console.log(`Estado de la matrícula: ${estado}`);

XML al servicio de HTML, CSS y JavaScript

XML puede ser la fuente de datos que alimenta tu interfaz web. Define la información una sola vez y reutilízala en todas las capas: HTML para la estructura, CSS para el estilo y JavaScript para renderizar y filtrar.

Repositorio de datos

Centraliza horarios, noticias o módulos en un XML validado por XSD

Transformación a HTML

Usa XSLT o JS para crear tarjetas, tablas o listados accesibles

Estilos reutilizables

CSS aplica branding y estados según atributos del XML

datos/modulos.xml

<?xml version="1.0" encoding="UTF-8"?>
<plan formacion="DAM" curso="2025">
    <modulo nombre="Programación" horas="256" estado="activo" 
            profesor="Ana López"/>
    <modulo nombre="Interfaces" horas="140" estado="nuevo" 
            profesor="Luis Romero"/>
</plan>

js/modulos.js

const contenedor = document.querySelector("[data-modulos]");

fetch("datos/modulos.xml")
    .then(res => res.text())
    .then(texto => new DOMParser().parseFromString(texto, "application/xml"))
    .then(xml => {
        const tarjetas = [...xml.querySelectorAll("modulo")].map(nodo => `
            <article class="modulo-card" 
                     data-estado="${nodo.getAttribute("estado")}">
                <h3>${nodo.getAttribute("nombre")}</h3>
                <p>${nodo.getAttribute("horas")} horas con 
                   ${nodo.getAttribute("profesor")}</p>
            </article>
        `);
        contenedor.innerHTML = tarjetas.join("");
    });

Proyecto Integrador

Pasos del proyecto

  1. Diseña el XSD: define tipos complejos para módulos, profesores y horarios.
  2. Crea el XML: agrega al menos 5 módulos con atributos como horas, curso y aula.
  3. Transforma con XSLT: genera una tabla HTML con los módulos y exporta también un resumen en texto.
  4. Exporta a JSON: utiliza un script (JavaScript o Python) que convierta el XML a JSON para una API simulada.
  5. Documenta: explica cómo consumir el XML en backend y cómo mostrarlo en el front.
