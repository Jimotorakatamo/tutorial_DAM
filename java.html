Java es un lenguaje de programación robusto, orientado a objetos y de alto rendimiento. En este tutorial, exploraremos desde los fundamentos hasta las características más modernas que lo mantienen relevante en el desarrollo de software a gran escala.

Introducción a Java

Java es un lenguaje de programación de alto nivel, orientado a objetos, y una plataforma informática que fue lanzada por primera vez por Sun Microsystems en 1995. Es conocido por su filosofía "Write Once, Run Anywhere" (WORA), lo que significa que el código compilado de Java puede ejecutarse en todas las plataformas que soporten Java sin necesidad de recompilación.

public class HolaMundo {
    public static void main(String[] args) {
        System.out.println("¡Hola, Mundo desde Java!");
    }
}

¡A Practicar!

Instala un JDK (Java Development Kit) si no lo tienes. Crea un archivo HolaMundo.java, copia el código de ejemplo, compílalo (javac HolaMundo.java) y ejecútalo (java HolaMundo) desde tu terminal.

Sintaxis Básica

Java tiene una sintaxis estructurada y fuertemente tipada, similar a C++. Cada aplicación tiene al menos una clase y un método main.

HolaMundo.java 
// Toda aplicación Java necesita una clase
public class HolaMundo {
    // El punto de entrada de la aplicación
    public static void main(String[] args) {
        // Imprime un mensaje en la consola
        System.out.println("¡Hola, Java!");
    }
}

Estructuras de Control

Las estructuras de control dirigen el flujo de tu programa, permitiendo tomar decisiones y repetir acciones.

ControlDeFlujo.java 
int puntuacion = 85;

// Estructura if-else if-else
if (puntuacion >= 90) {
    System.out.println("Sobresaliente");
} else if (puntuacion >= 70) {
    System.out.println("Notable");
} else {
    System.out.println("Aprobado");
}

// Bucle for
for (int i = 1; i <= 5; i++) {
    System.out.println("Iteración número: " + i);
}

// Bucle while
int contador = 0;
while (contador < 3) {
    System.out.println("Contador: " + contador);
    contador++;
}

// Switch (a partir de Java 14 con sintaxis mejorada)
String dia = "Lunes";
String tipoDia = switch (dia) {
    case "Lunes", "Martes", "Miércoles", "Jueves", "Viernes" -> "Día laborable";
    case "Sábado", "Domingo" -> "Fin de semana";
    default -> "Día inválido";
};
System.out.println(dia + " es un " + tipoDia);

¡A Practicar!

Clases y Objetos

La Programación Orientada a Objetos (POO) es el núcleo de Java. Las clases son plantillas para crear objetos.

Coche.java 
public class Coche {
    // Atributos (estado)
    String marca;
    String modelo;
    int anio;

    // Constructor para inicializar objetos
    public Coche(String marca, String modelo, int anio) {
        this.marca = marca;
        this.modelo = modelo;
        this.anio = anio;
    }

    // Métodos (comportamiento)
    public void acelerar() {
        System.out.println("El coche " + marca + " está acelerando.");
    }

    public String getInfo() {
        return "Coche: " + anio + " " + marca + " " + modelo;
    }
    
    public static void main(String[] args) {
        // Creación de un objeto (instancia) de la clase Coche
        Coche miCoche = new Coche("Toyota", "Corolla", 2022);
        
        // Llamada a sus métodos
        System.out.println(miCoche.getInfo());
        miCoche.acelerar();
    }
}

¡A Practicar!

Crea una clase Perro con una propiedad nombre y un método ladrar() que imprima "Guau!". Luego, crea un objeto de tipo Perro y llama a su método ladrar().

T2: Introducción a la Orientación a Objetos

2.1. Clases: atributos, métodos y visibilidad

Clase: Es el molde o plantilla que define las propiedades (atributos) y comportamientos (métodos) de los objetos.

Atributos: Variables que representan las características (ej: color, marca en una clase Coche).

Métodos: Acciones que la clase puede realizar (ej: acelerar(), frenar()).

Visibilidad: Controla el acceso: public (accesible por todos), private (solo en la clase), protected (clase y herederas).

2.2. Objetos y Mensajes

Un objeto es una instancia de una clase. Tiene:

  • Estado: Los valores de sus atributos en un momento dado.
  • Comportamiento: Lo que puede hacer (sus métodos).
  • Identidad: Cada objeto es único.

Los objetos interactúan enviando mensajes (llamadas a métodos).

2.3. Encapsulado

Principio que consiste en ocultar los detalles internos de una clase y exponer solo lo necesario a través de métodos. Esto protege los datos y permite modificar la implementación sin afectar al resto del programa.

2.4. Relaciones entre clases

Las clases pueden interactuar de varias formas:

  • Asociación: Una clase usa a otra (un Profesor enseña a un Alumno).
  • Agregación: Una clase contiene a otras que pueden existir por separado (una Universidad tiene Facultades).
  • Composición: Una clase contiene a otras que dependen de ella (un Coche tiene un Motor).
  • Herencia: Una clase (subclase) hereda atributos y métodos de otra (superclase).

2.5. Principios básicos de la POO

Abstracción

Identificar lo esencial de un objeto y representarlo en una clase.

Encapsulación

Ocultar la implementación y exponer solo lo necesario.

Herencia

Reutilizar código creando nuevas clases a partir de existentes.

Polimorfismo

Permitir que un mismo método tenga diferentes comportamientos.

T3: Elementos de un Programa

3.1. Estructura y Bloques Fundamentales

Un programa se compone de bloques fundamentales: Entrada (recibir datos), Proceso (transformarlos) y Salida (mostrar resultados).

3.2. Identificadores y Palabras Reservadas

Identificadores: Nombres que asignamos a variables, funciones, clases, etc. (ej: edad, nombreUsuario).

Palabras Reservadas: Términos propios del lenguaje con un significado especial que no podemos usar como identificadores (ej: if, class, public).

3.4. Variables

Una variable es un espacio en memoria con un nombre que almacena un valor que puede cambiar. Requiere:

  • Declaración: Informar de su existencia (int edad;).
  • Inicialización: Asignarle un valor inicial (edad = 18;).
  • Utilización: Usarla en operaciones (total = precio * cantidad;).

3.5. Tipos de Datos y Literales

Tipos de Datos: Definen la naturaleza del valor que una variable puede contener.

Primitivos

Básicos como int, double, boolean.

Compuestos

Complejos como String, Arrays o Clases.

Literales: Valores fijos escritos en el código, como 10, "Hola", o true.

3.8. Operadores y Expresiones

Operadores: Símbolos que realizan operaciones (+, -, *, /, ==, &&).

Expresión: Combinación de variables, literales y operadores que produce un valor (ej: (a + b) * c).

T4: Utilización de Objetos

4.2. Constructores

Son métodos especiales que se ejecutan al crear un objeto para inicializar sus atributos. Tienen el mismo nombre que la clase.

public class Coche {
    String color;
    // Constructor
    public Coche(String c) {
        this.color = c;
    }
}

4.3. Instanciación de Objetos

Crear un objeto (instancia) se hace con la palabra clave new, que reserva memoria e invoca al constructor.

// Declaración y creación en una sola línea
Coche miCoche = new Coche("Rojo");

4.7. Memoria: Tipos Básicos vs Objetos

Tipos básicos (primitivos): Almacenan el valor directamente en la memoria (ej: int x = 5;).

Objetos: Almacenan una referencia (una dirección de memoria) que apunta al lugar donde se encuentra el objeto real.

T5: Uso de Estructuras de Control

Las estructuras de control permiten modificar el flujo de ejecución de un programa para tomar decisiones o repetir acciones.

Selección

if-else, switch. Ejecutan código basado en condiciones.

Repetición

for, while, do-while. Repiten un bloque de código.

Salto

break, continue, return. Alteran el flujo normal de los bucles o métodos.

T6: Desarrollo de Clases

6.7. Sobrecarga de Métodos

Permite definir varios métodos con el mismo nombre pero con diferentes parámetros (distinto número o tipo). Esto ofrece flexibilidad al llamar al método.

class Calculadora {
    int sumar(int a, int b) {
        return a + b;
    }
    // Sobrecarga con tipo double
    double sumar(double a, double b) {
        return a + b;
    }
}

6.9. Paso de Parámetros

Por valor (primitivos): Se pasa una copia del valor. Los cambios dentro del método no afectan a la variable original.

Por referencia (objetos): Se pasa la dirección de memoria del objeto. Los cambios en los atributos del objeto dentro del método sí afectan al objeto original.

T7: Estructuras de Almacenamiento

7.2. Arrays

Un array es una colección de elementos del mismo tipo con un tamaño fijo. Se accede a los elementos mediante un índice que empieza en 0.

// Declaración, creación e inicialización de un array
int[] edades = {18, 20, 25, 30};

// Acceso a un elemento
int primeraEdad = edades[0]; // 18

// Recorrido con un bucle for-each
for (int edad : edades) {
    System.out.println(edad);
}

7.3. Cadenas de Caracteres (String)

En Java, las cadenas de texto son objetos de la clase String. Son inmutables, lo que significa que no pueden ser modificadas una vez creadas.

String saludo = "Hola";
String nombre = "Mundo";

// Concatenación
String mensaje = saludo + ", " + nombre + "!"; // "Hola, Mundo!"

// Métodos útiles
int longitud = mensaje.length(); // 13
char primerCaracter = mensaje.charAt(0); // 'H'
String subcadena = mensaje.substring(6, 11); // "Mundo"
boolean contiene = mensaje.contains("Mundo"); // true

T8: Utilización Avanzada de Clases

8.2. Herencia

La herencia permite que una clase (subclase) adquiera los atributos y métodos de otra (superclase), fomentando la reutilización de código. Se usa la palabra clave extends.

8.7. Clases Abstractas e Interfaces

Clases Abstractas: No se pueden instanciar. Sirven como plantillas para otras clases. Pueden tener métodos abstractos (sin implementación) que las subclases deben definir.

Interfaces: Definen un contrato de métodos que una clase debe implementar. Una clase puede implementar múltiples interfaces, lo que permite una forma de "herencia múltiple" de comportamiento.

8.9. Polimorfismo

El polimorfismo permite que un objeto de una subclase sea tratado como un objeto de su superclase. Esto, combinado con la sobreescritura de métodos, hace que una misma llamada a un método pueda tener diferentes comportamientos según el objeto real.

T9: Control y Manejo de Excepciones

Una excepción es un error que ocurre durante la ejecución y que interrumpe el flujo normal del programa. Java proporciona un mecanismo robusto para manejarlas.

9.3. Manejo de Excepciones (try-catch-finally)

Para manejar errores de forma controlada, se utiliza el bloque try-catch.

  • try: Contiene el código que podría lanzar una excepción.
  • catch: Captura y maneja la excepción si ocurre.
  • finally: (Opcional) Se ejecuta siempre, haya o no una excepción. Es ideal para liberar recursos.
try {
    int resultado = 10 / 0; // Esto lanzará una ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Error: No se puede dividir por cero.");
} finally {
    System.out.println("El bloque finally siempre se ejecuta.");
}

¡A Practicar!

Escribe un bloque try-catch que intente acceder a un índice inválido de un array (por ejemplo, el índice 5 en un array de 3 elementos). En el bloque catch, captura la excepción ArrayIndexOutOfBoundsException y muestra un mensaje amigable.

T10: Colecciones de Datos

Las colecciones son estructuras dinámicas para almacenar y manipular grupos de objetos. Son más flexibles que los arrays.

Listas (List)

Conjunto ordenado de elementos que pueden repetirse (ArrayList).

Pilas (Stack)

Estructura LIFO (Último en entrar, primero en salir).

Colas (Queue)

Estructura FIFO (Primero en entrar, primero en salir).

Mapas (Map)

Colección de pares clave-valor (HashMap).

EjemploColecciones.java 
import java.util.ArrayList;
import java.util.HashMap;

// Lista
ArrayList<String> frutas = new ArrayList<>();
frutas.add("Manzana");

// Mapa
HashMap<String, Integer> edades = new HashMap<>();
edades.put("Juan", 25);

T11: Introducción a Streams

Los streams en Java son secuencias de datos que permiten procesar colecciones de manera funcional. Se introdujeron en Java 8 y son parte del paquete java.util.stream.

EjemploStreams.java 
import java.util.Arrays;
import java.util.List;

List<String> nombres = Arrays.asList("Ana", "Luis", "Alberto");
nombres.stream()
       .filter(nombre -> nombre.startsWith("A"))
       .forEach(System.out::println); // Imprime Ana, Alberto

¡A Practicar!

T12: Expresiones Lambda

Las expresiones lambda son una forma concisa de representar funciones anónimas en Java. Se utilizan principalmente para implementar interfaces funcionales (interfaces con un solo método abstracto).

EjemploLambdas.java 
// Interfaz funcional
interface Operacion {
    int calcular(int a, int b);
}

// Usando una expresión lambda
Operacion suma = (a, b) -> a + b;
int resultado = suma.calcular(5, 3); // 8

¡A Practicar!

Crea una expresión lambda que tome un String y devuelva su longitud. Puedes usar la interfaz funcional Function<String, Integer>.

T13: Herramientas de Construcción (Maven y Gradle)

Maven y Gradle son herramientas de automatización de compilación y gestión de dependencias en proyectos Java.

13.1. Maven

Maven utiliza un archivo pom.xml para definir la configuración del proyecto, sus dependencias, y los plugins a utilizar.

<!-- Dependencia de JUnit 5 en pom.xml (Maven) -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter-api</artifactId>
    <version>5.8.2</version>
    <scope>test</scope>
</dependency>

13.2. Gradle

Gradle utiliza un archivo build.gradle donde se define el proyecto usando una sintaxis basada en Groovy o Kotlin.

// Ejemplo de build.gradle
plugins {
    id 'java'
}

repositories {
    mavenCentral()
}

dependencies {
    testImplementation 'org.junit.jupiter:junit-jupiter-api:5.8.2'
}

¡A Practicar!

Si tienes Maven o Gradle instalado, intenta crear un nuevo proyecto. Busca en el repositorio de Maven (Maven Central) la dependencia de "Google Gson" y añádela a tu archivo de configuración.

T14: Introducción a Spring Boot

Spring Boot es un framework para crear aplicaciones Java basadas en Spring de manera rápida y sencilla. Proporciona configuraciones predeterminadas y un conjunto de herramientas para desarrollar aplicaciones listas para producción.

Controlador.java 
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class ControladorHola {

    @GetMapping("/")
    public String hola() {
        return "¡Hola desde Spring Boot!";
    }
}

¡A Practicar!
