Aprende XML

XML (eXtensible Markup Language) es un lenguaje de marcado diseñado para almacenar y transportar datos. Es flexible, legible y ampliamente utilizado en la web y aplicaciones empresariales.

Sintaxis y Reglas Fundamentales

Un documento XML "bien formado" debe seguir reglas estrictas para ser procesado correctamente.

  • Debe tener un único elemento raíz.
  • Todas las etiquetas deben tener su correspondiente etiqueta de cierre.
  • Las etiquetas son sensibles a mayúsculas y minúsculas (<Libro> es diferente de <libro>).
  • Los elementos deben estar anidados correctamente, sin solaparse.
  • Los valores de los atributos deben estar siempre entre comillas (simples o dobles).

Validación: DTD y XSD

Para que un documento XML sea "válido", debe cumplir con una estructura definida, ya sea mediante un DTD o un XML Schema (XSD).

DTD (Document Type Definition)

Define la estructura legal y los elementos de un documento XML. Es la forma más antigua de validación.

XSD (XML Schema Definition)

Una alternativa más potente y flexible a DTD. Soporta tipos de datos, restricciones y espacios de nombres.

XPath: Consultando en XML

XPath es un lenguaje que permite navegar a través de los elementos y atributos de un documento XML para seleccionar nodos específicos.

/biblioteca/libro[@categoria='ficcion']  // Selecciona todos los libros de ficción
//libro[precio > 25]                   // Selecciona libros con precio mayor a 25
//libro/titulo                         // Selecciona todos los títulos de los libros

XSLT: Transformando XML

XSLT (eXtensible Stylesheet Language Transformations) se usa para transformar un documento XML en otro formato, como HTML, PDF o texto plano.

<xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform">
    <xsl:template match="/">
        <html><body>
            <h2>Mis Libros</h2>
            <xsl:for-each select="biblioteca/libro">
                <p><xsl:value-of select="titulo"/> por <xsl:value-of select="autor"/></p>
            </xsl:for-each>
        </body></html>
    </xsl:template>
</xsl:stylesheet>

Namespaces (Espacios de Nombres)

Los namespaces se utilizan para evitar conflictos de nombres entre elementos que se llaman igual pero tienen diferente significado. Se definen con el atributo xmlns.

<root>
    <h:table xmlns:h="http://www.w3.org/TR/html4/">
        <h:tr><h:td>Celda HTML</h:td></h:tr>
    </h:table>

    <f:table xmlns:f="https://www.w3schools.com/furniture">
        <f:name>Mesa</f:name>
        <f:width>80</f:width>
    </f:table>
</root>
